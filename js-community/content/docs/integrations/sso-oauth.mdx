# SSO & OAuth

JS Community supports social sign-in via Google and GitHub, powered by [better-auth](https://www.better-auth.com/). Both providers are optional — email/password login works independently.

## How it works

1. The user clicks "Continue with Google" or "Continue with GitHub" on the login page
2. The browser is redirected to `GET /api/auth/sign-in/[provider]`
3. better-auth redirects to the provider's OAuth authorization URL
4. The user grants permission in the provider's UI
5. The provider redirects back to `GET /api/auth/callback/[provider]`
6. better-auth exchanges the code for an access token, creates or retrieves the user account, and sets a session cookie

## Google OAuth

### 1. Create a Google OAuth app

1. Go to the [Google Cloud Console](https://console.cloud.google.com)
2. Create a new project (or select an existing one)
3. Navigate to **APIs & Services → Credentials**
4. Click **Create Credentials → OAuth client ID**
5. Application type: **Web application**
6. Add authorized redirect URIs:
   - `http://localhost:3000/api/auth/callback/google` (development)
   - `https://your-domain.com/api/auth/callback/google` (production)
7. Copy the **Client ID** and **Client Secret**

### 2. Set environment variables

```bash
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
```

### 3. Verify the configuration

Navigate to `/login` in development. The "Continue with Google" button appears only when both variables are set.

## GitHub OAuth

### 1. Create a GitHub OAuth app

1. Go to **Settings → Developer settings → OAuth Apps** in GitHub
2. Click **New OAuth App**
3. Set the **Authorization callback URL** to:
   - `http://localhost:3000/api/auth/callback/github` (development)
   - `https://your-domain.com/api/auth/callback/github` (production)
4. Copy the **Client ID** and generate a **Client Secret**

### 2. Set environment variables

```bash
GITHUB_CLIENT_ID=...
GITHUB_CLIENT_SECRET=...
```

## Account linking

better-auth links OAuth accounts by email address. If a user already registered with email/password using the same email as their Google account, signing in with Google will attach to the existing account.

OAuth accounts are stored in the `accounts` table:

| Column | Description |
|--------|-------------|
| `providerId` | `"google"` or `"github"` |
| `accountId` | The user's ID within the provider |
| `accessToken` | OAuth access token (not persisted long-term) |
| `userId` | FK to the `users` table |

## Session management

Sessions created via OAuth follow the same lifetime and cookie configuration as email/password sessions. Expired sessions are cleaned up daily by the `cleanup-expired-sessions` Inngest function.

## Adding more providers

better-auth supports many additional OAuth providers (Twitter/X, Discord, LinkedIn, etc.). To add one:

1. Register an app with the provider and get client ID/secret
2. Add the provider to `src/lib/auth.ts`:

```typescript
socialProviders: {
  discord: {
    clientId: process.env.DISCORD_CLIENT_ID || "",
    clientSecret: process.env.DISCORD_CLIENT_SECRET || "",
    enabled: !!(process.env.DISCORD_CLIENT_ID && process.env.DISCORD_CLIENT_SECRET),
  },
},
```

3. Add login buttons in the login and register pages

See the [better-auth documentation](https://www.better-auth.com/docs/authentication/social-sign-on) for the full list of supported providers.
