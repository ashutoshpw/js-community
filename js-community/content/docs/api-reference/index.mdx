# API Reference

JS Community exposes a REST API under `/api`. All endpoints return JSON. Authenticated endpoints require a valid session cookie obtained by signing in at `/api/auth/sign-in`.

## Base URL

```
https://your-domain.com/api
```

## Authentication

Most write endpoints and all admin endpoints require authentication. Pass the session cookie automatically from the browser, or include it explicitly in server-side requests.

Unauthenticated requests to protected endpoints return:

```json
{ "error": "Unauthorized" }
```

with HTTP status `401`.

## Standard response shape

Successful responses return the relevant resource or collection directly:

```json
{ "topic": { ... } }
{ "topics": [...], "total": 120, "page": 1, "perPage": 20 }
```

Error responses always include an `error` string:

```json
{ "error": "Topic not found" }
```

## Rate limiting

See [Security](/docs/admin-guide/security) for rate limit thresholds. Exceeded limits return `429` with a `Retry-After` header.
