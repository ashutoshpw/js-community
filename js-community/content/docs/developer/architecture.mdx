# Architecture

## Monorepo layout

```
js-community/                  ← repo root
├── js-community/              ← Next.js application
│   ├── src/
│   │   ├── app/               ← Next.js App Router (routes + components)
│   │   ├── db/schema/         ← Drizzle ORM schema definitions
│   │   ├── inngest/           ← Inngest background functions
│   │   ├── lib/               ← Shared utilities and services
│   │   └── test/              ← Test setup, utils, mocks
│   ├── content/docs/          ← MDX documentation files (Fumadocs)
│   └── public/                ← Static assets
├── discourse/                 ← Legacy Discourse reference (submodule)
└── scripts/                   ← Shared build/CI scripts
```

## Server vs Client components

Next.js App Router defaults to **Server Components**. Use `"use client"` only when you need:

- Browser APIs (`window`, `document`, `localStorage`)
- React state (`useState`, `useReducer`)
- React effects (`useEffect`, `useCallback`)
- Event handlers (`onClick`, `onChange`)
- Third-party client-only libraries (framer-motion, etc.)

### Conventions in this codebase

| Pattern | Example |
|---------|---------|
| Page file is a Server Component | `src/app/forum/t/[id]/[slug]/page.tsx` |
| Interactive parts are split into a `*Client.tsx` file | `ReviewClient.tsx`, `SettingsClient.tsx` |
| Data fetching happens in the Server Component | DB queries in `page.tsx` |
| Props are passed down to the Client Component | `<ReviewClient flags={flags} />` |

This keeps the bundle small — only code that truly needs the browser ships as client JS.

## Library structure (`src/lib/`)

| File/Dir | Purpose |
|----------|---------|
| `auth.ts` | better-auth configuration |
| `auth-client.ts` | Client-side better-auth instance |
| `auth-helpers.ts` | `isUserAdmin`, `isUserModerator` utilities |
| `database.ts` | Drizzle ORM connection |
| `email.ts` | Resend email templates and sending functions |
| `inngest.ts` | Inngest client instance |
| `rate-limit.ts` | Password-reset rate limiting (Postgres-backed) |
| `permissions/` | Trust level system and permission checks |
| `realtime/` | SSE event store and types |
| `markdown.ts` | marked.js rendering pipeline |
| `sanitize.ts` | DOMPurify sanitization |
| `validation.ts` | Email, password, username validation |
| `logger.ts` | Structured logger |
| `env.ts` | Environment variable validation |
| `cache.ts` | Postgres-backed query cache |
